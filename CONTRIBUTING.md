# Contributing

Thank you for helping improve OmniGuard.

1. Discuss substantial API or security-behavior changes in an issue first.
2. Create a focused branch and keep unrelated formatting out of the change.
3. Add tests for new behavior and for every bug fix.
4. Run `npm ci`, `npm test`, `npm run build`, and `npm pack --dry-run`.
5. Update the README, security guarantees, and changelog when public behavior changes.

Security fixes should be reported privately according to [SECURITY.md](./SECURITY.md), not submitted as a public pull request before disclosure is coordinated.
